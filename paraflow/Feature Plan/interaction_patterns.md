# CareerMatch AI 交互设计模式

定义平台中各种操作的交互方式，明确何时使用模态框、何时使用页面跳转。

## 🎯 **交互设计原则**

### **模态框 (Modal) 使用场景**
适用于：
- 简单、快速的操作
- 不需要丢失当前页面上下文
- 操作结果需要立即反馈
- 内容较少，不需要滚动

### **页面跳转使用场景**
适用于：
- 复杂、多步骤的操作
- 需要专注的完整界面
- 大量内容需要展示
- 需要SEO和独立URL访问

## 📋 **具体交互设计规范**

### **1. 认证相关**

| 功能 | 交互方式 | 触发场景 | 说明 |
|------|---------|---------|------|
| **快速登录** | Modal | 从任何页面点击"登录" | 保持当前页面上下文 |
| **完整登录** | Page | 直接访问登录URL | 独立页面，可分享链接 |
| **用户注册** | Page | 点击"注册"按钮 | 复杂表单，需要专注填写 |
| **忘记密码** | Modal → Page | 登录页面点击 | 简单输入用模态框，完整流程用页面 |
| **登出确认** | Modal | 点击"退出登录" | 简单确认操作 |

### **2. 简历管理**

| 功能 | 交互方式 | 触发场景 | 说明 |
|------|---------|---------|------|
| **创建新简历** | Page | 简历管理中心 | 需要选择模板和填写信息 |
| **快速编辑简历** | Modal | 列表页面快速修改 | 修改名称、状态等简单信息 |
| **详细编辑简历** | Page | 点击"编辑"按钮 | 复杂的WYSIWYG编辑界面 |
| **删除简历确认** | Modal | 点击删除按钮 | 简单确认操作 |
| **复制简历** | Modal | 点击复制按钮 | 输入新名称确认 |
| **导出简历选项** | Modal | 点击导出按钮 | 选择格式和选项 |

### **3. 岗位管理**

| 功能 | 交互方式 | 触发场景 | 说明 |
|------|---------|---------|------|
| **添加岗位** | Modal | 岗位管理页面 | 简单URL输入或手动添加 |
| **批量导入岗位** | Page | 高级添加功能 | 复杂的批量操作界面 |
| **查看岗位详情** | Page | 点击岗位卡片 | 完整的详情和分析界面 |
| **快速更新状态** | Modal | 状态下拉菜单 | 简单的状态选择 |
| **批量操作** | Modal | 选中多个岗位 | 批量状态更新、删除等 |
| **筛选设置** | Sidebar | 筛选控制面板 | 侧边栏滑出，不遮挡主内容 |

### **4. AI功能相关**

| 功能 | 交互方式 | 触发场景 | 说明 |
|------|---------|---------|------|
| **生成定制简历** | Modal → Page | 岗位详情页面 | 确认参数用模态框，生成过程用页面 |
| **查看匹配分析** | Page | 岗位详情标签页 | 复杂的9维度分析需要完整页面 |
| **AI优化建议** | Sidebar | 简历编辑器 | 右侧面板形式，不干扰编辑 |
| **重新分析** | Modal | 分析结果页面 | 简单的重新处理确认 |

### **5. 通知和反馈**

| 功能 | 交互方式 | 触发场景 | 说明 |
|------|---------|---------|------|
| **成功消息** | Toast | 操作完成后 | 右上角临时通知 |
| **错误提示** | Modal | 操作失败时 | 需要用户确认的错误信息 |
| **确认操作** | Modal | 删除、重要变更 | 简单的是/否确认 |
| **进度显示** | Modal | 长时间操作 | AI分析、文档生成等 |

### **6. 设置和配置**

| 功能 | 交互方式 | 触发场景 | 说明 |
|------|---------|---------|------|
| **个人资料编辑** | Page | 用户菜单点击 | 复杂的表单需要专门页面 |
| **快速设置修改** | Modal | 设置页面单项修改 | 简单的开关、选择操作 |
| **求职偏好设置** | Page | 档案设置子页面 | 复杂的多项配置 |
| **通知偏好** | Inline | 设置页面内 | 直接在页面内切换，无需模态框 |

## 🎨 **视觉设计规范**

### **模态框设计**
- **尺寸**：
  - 小型：400px 宽（确认操作）
  - 中型：600px 宽（表单填写）
  - 大型：800px 宽（复杂内容）
- **位置**：垂直水平居中
- **背景**：半透明黑色遮罩 (rgba(0,0,0,0.5))
- **动画**：淡入淡出 + 轻微缩放

### **侧边栏设计**
- **筛选面板**：从右侧滑出，320px 宽
- **AI建议面板**：固定右侧，280px 宽
- **导航菜单**：从左侧滑出（移动端），250px 宽

### **页面过渡**
- **标准跳转**：无动画，直接加载
- **返回操作**：支持浏览器后退
- **深度链接**：所有重要页面支持直接URL访问

## 📱 **移动端适配**

### **移动端特殊规则**
- **模态框**：在小屏幕上变为全屏覆盖
- **侧边栏**：统一从底部向上滑出
- **多步骤操作**：使用步进器指示进度
- **长表单**：分页或分步骤处理

## 🔄 **状态管理**

### **加载状态**
- **页面级加载**：骨架屏 + 进度指示
- **组件级加载**：Spinner + 禁用状态
- **数据更新**：乐观更新 + 错误回滚

### **错误处理**
- **网络错误**：Toast通知 + 重试按钮
- **验证错误**：表单内联提示
- **系统错误**：模态框详细说明 + 联系支持
